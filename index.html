Простой фреймворк для PHP

mini_framework представляет из себя простой MVC фреймворк для создание веб приложений.

<?php

/**
 * Данная функция вызывается до вызова 
 * опредленного действия в этом же контроле
 */
function actions_init () {
    
}

/**
 * Главное действие
 */
function action_index () {
    layout('index');
}

Фичи

Обязательно нужно написать модульные, функциональные и приемочные тесты. Ну и админу надо десктоп приличный. Это как синхронный цикл в Meteor. Главное не забыть о бекапах.

  • По-моему архитектура этого проекта очевидна.

    Когда проект разрастется запилим свой QueryBuilder. Для начала перезагрузим консоль. Это как синхронный цикл в Meteor. Необходимо будет автоматически скейлить облачные серверы. А вообще все это можно сделать на одних callback'ах.

  • Я бы делал так.

    Вынесем в отдельный слой DataMapper, Identity Map и все связанные объекты. Возьмем какой нибудь графический тулкит. Когда проект разрастется запилим свой QueryBuilder. Все запросы будут отправляться с помощью JS. Ну а если сильно не заморачиваться то это все можно на процедурке написать.

  • Все очень просто.

    Если для конкретной платформы – С++. Обязательно нужно написать модульные, функциональные и приемочные тесты. Проработаем наилучшую архитектуру для back-end коннектора к различным API. Повесим observer на каждый инстанс. В общем в энтерпрайзе делают так.

Документация

Нужно обязательно продумать структуру приложения. Перепишем foreach'и на while. Для начала перезагрузим консоль. И что бы ни одного onclick'a в HTML!
